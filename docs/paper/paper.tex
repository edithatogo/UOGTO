\begin{abstract}
Game-theoretic models increasingly move between mathematical analysis, multi-agent reinforcement learning, mechanism design, security, simulation, and tool-using AI systems. Yet the objects that make those models interpretable, including players, actions, strategies, information states, payoffs, mechanisms, outcomes, sessions, and traces, are still usually exchanged as prose, code, diagrams, or project-specific tables. That makes it difficult to compare models, validate examples, cite assumptions, or reuse evidence across domains. We present the Universal Open Game Theory Ontology (UOGTO), a modular RDF/OWL ontology for representing game-theoretic and simulation-facing semantics as inspectable knowledge graphs. UOGTO separates abstract game specifications from executed sessions and provenance traces; validates examples with SHACL; exposes competency questions in SPARQL; publishes accepted mappings in both Turtle and SSSOM; and packages source discovery, mapping review, quality metrics, figure data, SourceRight citation checks, and arXiv preflight evidence as version-controlled artefacts. In a comparative workflow spanning 21 source families, UOGTO normalized 4,037 term rows, generated 460 mapping candidates, and accepted 10 conservative alignments while preserving rejected and domain-review rows as negative evidence. UOGTO is therefore not presented as a final catalogue of all game theory. Its contribution is a reproducible semantic layer that makes strategic-interaction claims more explicit, testable, and reusable.
\end{abstract}

\section{Introduction}
Game theory supplies a compact language for strategic interaction, from the Prisoner's Dilemma \cite{rapoport-chammah-1965} to modern reinforcement-learning environments for games \cite{open-spiel-2019,arxiv-2006-06580v3}. The same conceptual vocabulary now appears in auctions, voting and social choice, security games, bargaining, mechanism design, agent-based simulation, digital twins, institutional analysis, and tool-using AI agents. The modelling media, however, have not converged. A payoff matrix, a simulation trace, a Python environment, a standards document, and a provenance graph may all describe games, but they rarely expose their assumptions in a shared form.

The practical consequence is not only inconvenience. Without a shared semantic layer, readers cannot easily ask which agents participated, which actions were legal, which strategy was observed, which payoff was asserted, which mechanism produced an outcome, or which trace supports a claim. The problem becomes sharper when game-theoretic claims are used as evidence in computational experiments, policy simulations, or AI-agent evaluations. A model may be executable, but its commitments may still be hard to inspect.

UOGTO addresses this gap by representing game-theoretic objects as linked data. It uses RDF as the graph substrate \cite{w3c-rdf11-concepts}, OWL 2 for reusable ontology structure \cite{w3c-owl2-overview}, SHACL for closed-world validation of examples \cite{w3c-shacl}, JSON-LD for web serialization \cite{w3c-json-ld11}, and SPARQL for competency questions \cite{w3c-sparql11-query}. The aim is not to replace formal game theory. The aim is to make the objects around formal and executable games explicit enough to validate, align, cite, and reuse.

This paper evaluates UOGTO by its modelling choices and by the evidence package that surrounds them. UOGTO therefore couples the ontology with source discovery records, mapping review data, SSSOM tables, SHACL examples, quality metrics, figure sources, SourceRight reference checks, and arXiv packaging gates. \section{Results}
\subsection{A semantic core separates games, executions, and evidence}
UOGTO models a game specification separately from a game instance, session, or trace. This distinction is the ontology's main design decision. A specification describes structural possibilities; a session records an occurrence; a trace records the evidence produced by execution. Strategies, actions, payoffs, outcomes, mechanisms, information states, and execution bindings are also represented as distinct modelling surfaces. The separation avoids a common ambiguity in computational game descriptions: the same word ``game'' can refer to an abstract form, a configured environment, or a particular run.

The current examples cover normal-form and extensive-form games, cooperative games, auctions, matching markets, voting and social choice, stochastic Markov games, security Stackelberg games, privacy-budget games, LLM tool-use games, Petri-net execution games, and digital-twin security games. These examples do real work. They are validation artefacts: each tests whether the vocabulary can support concrete game descriptions without collapsing specification, execution, and evidence into one object. Figure~\ref{fig:architecture} should present this separation as the opening visual.

\subsection{The evidence package makes ontology quality inspectable}
The ontology is accompanied by a module audit table covering labels, definitions, SHACL links, examples, competency questions, OWL profile status, and reasoner status. This table is generated in CSV, JSON, and Markdown under \texttt{docs/article-hardening/article-facing-tables/}. It gives reviewers a compact way to inspect whether UOGTO's coverage claims are supported by validation artefacts rather than by prose alone.

The quality metrics report complete label and definition coverage across the audited ontology entities, RDF parsing status, SHACL example validation, OWL-RL reasoner status, orphan-class checks, relation-richness measures, hierarchy depth, import depth, example coverage, and competency-query linkage. \subsection{Comparative mapping favours precision over broad equivalence}
The comparative workflow reviewed 21 source families relevant to game theory, simulation, provenance, time, web schemas, workflow modelling, and adjacent formalisms. It extracted 4,037 term rows across UOGTO and external sources, generated 460 deterministic mapping candidates, accepted 10 alignments, marked one candidate for domain review, and rejected 449 candidates. The low acceptance rate is deliberate. It prevents lexical similarity from becoming asserted equivalence.

Accepted alignments are emitted as Turtle and SSSOM. Rejected and domain-review rows remain visible in the review data. This makes the mapping package auditable: a reader can see not only what UOGTO aligns to, but also what it chose not to assert. Figure~\ref{fig:mapping-flow} should show the candidate-to-review-to-alignment flow. Figure~\ref{fig:evidence-heatmap} should show source-family evidence levels so that parsed RDF sources are not conflated with metadata-only or literature-only evidence.

\subsection{Network analyses show bridges, not a universal merger}
The network analysis identified 74 source graph nodes, 10 term-alignment bipartite edges, 391 source-similarity edges, and 11 UOGTO module-coverage edges. Central UOGTO modules included mean-field games, cooperative games, network games, congestion-routing games, evolutionary games, compositional open games, epistemic games, and causal games. These results support a restrained interpretation: UOGTO connects several modelling traditions, but it does not erase their differences.

Sensitivity analyses compare accepted-only mappings, accepted plus close/related mappings, and scenarios with metadata-only sources excluded. These analyses are important for claim discipline. If a bridge disappears when metadata-only sources are removed, it should be described as documentary evidence rather than structural alignment. Figure~\ref{fig:network-sensitivity} should therefore focus on robust bridges and clearly mark evidence limits.

\subsection{Publication hardening turns the ontology into a reusable research object}
UOGTO is packaged as more than a vocabulary file. The release includes citation metadata, DOI and registry surfaces, WIDOCO documentation, SourceRight reference checks, arXiv source-package cleaning, a source-leak/privacy audit manifest, RO-Crate metadata, DuckDB/tabular artefact design, and CI validation. The article-hardening protocol records source discovery, negative evidence, dual-screening samples, reviewer calibration, case studies, and structured outputs. Figure~\ref{fig:reproducibility} should show this chain from ontology source to validated examples, reviewed mappings, article tables, and submission gates.

This packaging changes the nature of the contribution. UOGTO is not only a semantic proposal; it is a testable research object. Readers can inspect the terms, rerun validation, review mapping decisions, and trace manuscript claims to artefacts.

\section{Discussion}
UOGTO responds to a practical problem in contemporary game-theoretic work: strategic-interaction models now travel across disciplines faster than their assumptions can be inspected. A common ontology cannot decide which equilibrium concept is appropriate, whether a mechanism is normatively justified, or whether a simulation is realistic. It can, however, expose the entities and relations that make those questions answerable.

The design choice that matters most is restraint. UOGTO keeps specifications separate from executions, and candidate mappings separate from accepted alignments. That is less dramatic than claiming comprehensive coverage, but it is more useful for reproducible research. It allows a paper, simulation, or agent evaluation to state exactly what was modelled and what evidence supports the statement.

The second contribution is methodological rather than taxonomic. Ontology papers often describe vocabularies as if the hierarchy alone were the object of review. UOGTO instead treats the hierarchy, examples, shapes, competency questions, mappings, source records, quality metrics, figures, and submission checks as one evidence package. That package can be used by game theorists, simulation researchers, ontology engineers, and AI researchers without asking each group to adopt the same software stack.

The next step is selective expansion rather than indiscriminate growth. The missing-element disposition table is designed for this purpose. Concepts should be added to UOGTO only when they improve reusable semantics; otherwise they should be aligned externally, deferred, rejected as duplicates, rejected as out of scope, or sent to domain review.

\section{Methods}
\subsection{Ontology representation and validation}
UOGTO is expressed in RDF and OWL 2, with JSON-LD contexts for web serialization and SHACL shapes for example validation \cite{w3c-rdf11-concepts,w3c-owl2-overview,w3c-json-ld11,w3c-shacl}. Competency questions are expressed in SPARQL \cite{w3c-sparql11-query}. Validation parses ontology modules, validates examples against SHACL, and executes competency queries. The portable baseline uses RDFLib and pySHACL; optional ROBOT-style reports are retained as an additional evidence layer when Java tooling is available.

\subsection{Source discovery and screening}
Source discovery follows a PRISMA-ScR-style article-hardening protocol. Search routes include public scholarly and metadata services such as OpenAlex, arXiv, and Crossref \cite{openalex,arxiv-api,crossref}, public ontology registries, standards documentation, and repository searches. Search events are recorded with query text, result count, evidence level, inclusion rationale, licence disposition, and reviewer handoff. Sources are classified as parsed RDF, structured non-RDF, metadata-only, literature-only, or excluded. Negative evidence searches are retained so that absence claims are visible.

\subsection{Mapping review and robustness}
The mapping pipeline normalizes terms, generates deterministic candidates from lexical, definitional, hierarchical, property-signature, type-compatibility, and source-reliability signals, and separates candidates from accepted assertions. Accepted mappings are exported in Turtle and SSSOM. Robustness analyses ablate exact labels, normalized labels, definition similarity, hierarchy context, property signatures, and embedding similarity. Reviewer calibration and adjudication summaries are stored beside the mapping outputs.

\subsection{Manuscript source integrity}
References are stored in CSL JSON and reconciled with LaTeX citation keys. SourceRight is used to validate the CSL file, generate reference-integrity reports, and check numeric citation exports against the manuscript citation surface. The arXiv source package is also audited for hidden files, private notes, auxiliary files, unused figures, embedded metadata, credentials, and private URLs. These checks support source integrity; they are not evidence that every cited source endorses UOGTO.

\section{Limitations}
UOGTO has four main limitations. First, metadata-only sources cannot support the same claims as parsed RDF artefacts. The manuscript and supplement therefore distinguish parsed evidence from documentary evidence. Second, accepted mappings are intentionally sparse. This favours precision, but it leaves many plausible relationships for future review. Third, figure and deck readiness depend on a frozen manuscript. If figure numbering, captions, data, or slide claims change, the scoring loop should be repeated. Fourth, UOGTO is broad but incomplete. New game-theory concepts should pass through the explicit disposition workflow before they are added.

\section{Conclusion}
UOGTO provides a semantic layer for game-theoretic and simulation-facing research that is both machine-actionable and reviewable. Its value is not final completeness. Its value is that games, sessions, traces, strategies, actions, payoffs, outcomes, mechanisms, mappings, and evidence can be represented, validated, aligned, and cited in a shared form. That makes strategic-interaction claims easier to inspect and easier to reuse.
