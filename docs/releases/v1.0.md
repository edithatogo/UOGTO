# UOGTO v1.0 Release Notes

## Summary
UOGTO v1.0 is the first publication-oriented release of the Universal Open Game Theory Ontology. It packages the validated ontology modules, SHACL shapes, JSON-LD contexts, examples, competency queries, and generated documentation for citation and public registry discovery.

## Repository
- GitHub: <https://github.com/edithatogo/UOGTO>
- Canonical namespace: `https://w3id.org/uogto/core#`
- Extension namespace: `https://w3id.org/uogto/extensions#`
- Preferred core prefix: `uogto`
- Preferred extension prefix: `uogtox`

## Included Assets
- `ontologies/core/`: Core game-theoretic concepts, properties, and execution-neutral semantics.
- `ontologies/extensions/`: Classical, cooperative, information, mechanism design, MARL, network, norms/contracts, LLM/digital twin, and executable binding modules.
- `ontologies/alignments/`: Schema.org, PROV-O, Open Games, MCP/A2A, and WoT alignment modules.
- `shapes/`: SHACL validation shapes.
- `jsonld/`: JSON-LD contexts.
- `examples/`: Turtle and JSON-LD examples.
- `competency-questions/`: SPARQL query checks.
- `dist/`: Generated merged ontology, shape, context, checksum, and release-manifest artifacts after `make release-assets`.

Release process:
1. Enable the `edithatogo/UOGTO` repository in Zenodo GitHub integration.
2. Confirm Zenodo is configured to archive GitHub releases.
3. Enable GitHub Pages in repository settings and set the repository variable `ENABLE_PAGES_DEPLOY=true`.
4. Create a GitHub release tagged `v1.0.0`.
5. Wait for Zenodo to archive the release and mint the DOI.
6. For account-side inspection without a browser, set `ZENODO_ACCESS_TOKEN` and run `python scripts/maintenance/check_zenodo_depositions.py --json`; use `--require-token --require-uogto` when a matching deposition is expected.
7. Run `python scripts/maintenance/check_doi_status.py --live --require-doi` to verify the recorded DOI resolves through Zenodo or the DOI resolver.
8. Run `python scripts/maintenance/record_zenodo_doi.py <doi>` to replace the DOI placeholder in this file and registry submission documents when minting a future release DOI.
