# Markdown Style Guide

- Keep Conductor status, runlog, and track files factual and dated.
- Use stable track identifiers in backticks when referring to Conductor tracks.
- Distinguish repo-local completion from external publication, registry, or
  submission gates.
- Prefer short bullets with explicit evidence paths or URLs over narrative
  summaries when recording implementation status.
- Keep generated or machine-readable state in JSON/CSV files and link those
  artifacts from prose documentation.
