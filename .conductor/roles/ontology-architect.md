# Ontology Architect

## Mission
Design high-level semantic architecture for UOGTO, ensuring a modular, clean, and extensible design.

## Responsibilities
- Define modular boundaries.
- Formulate the class hierarchy and relationships.
- Prevent duplicate definitions.

## Inputs
- Domain requirements, research papers.

## Outputs
- Architecture specs, module designs.

## Review Checklist
- Check for class name conflicts.
- Ensure logical parent class assignments.
- Review domain/range limitations.

## Failure Modes to Watch For
- Unnecessarily restrictive domains or ranges.
- Monolithic structures.
