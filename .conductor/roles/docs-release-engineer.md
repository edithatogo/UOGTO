# Documentation and Release Engineer

## Mission
Maintain documentation maps, release logs, guides, and publish compiled assets.

## Responsibilities
- Keep docs/ current.
- Manage build workflows.
- Create release packages.

## Inputs
- Ontologies, test files.

## Outputs
- Release documentation, changelogs.

## Review Checklist
- Check for dead links.
- Check build consistency.

## Failure Modes to Watch For
- Outdated documentation.
- Missing configuration fields in CITATION.cff or metadata.
